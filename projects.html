Buddies

A replica for the meetUp app currently in the market. It allows user to sign-in with google, create groups to have gatherings of people with similar interests. A user could join a group or create one and then have diffrent events. Joining a group also allows user to be part of group chat and track the live location of other group members. The privacy of users if maintained and information is only shared if they allow it.

Google login Api

Used the google login option for the user, which made it easier for us to get the basic profile information and also gave the user a sence of securtiy. With the use of firebase to store the token given by google to individual user, i used the token to implement the group chats those are stored and are real time. The token is also usedd to fetch the location of the user from where ever he signs in upon permision

Home page

Fetch the Information from google like user's name, user, photo and user's email to display on the home page and user's token to use it in the real time chatting and location along with configuration of groups

Creating a group

User has the option to create a new group and start his own meetup by adding information like location, description and hash tags. After creation of a group a user also could add the events to it and give information to it including, email, time, duration, and location.

Search Groups or Events

User could search a group based on its name, location, event and hash tags. Join it if seems interested. After joining the group, user also gets included in the group chat room (created with the use of real time NoSql database, Firebase).

Join a group

To join a group user needs to click on the Join The Group button, which will then add the group into their group list and hence they would be able to be a part of the group discussions

Firebase for Realtime chatting

Creating a structure in firebase to store the chats of the all the groups and also keep them in the same order as they were received. An option to store it along with the date could also be made which will also enable the user to search chats from a particular user and date.

Group chat

This is how the chat page looks like, the layout design is minimal for simplicity but to make it more user friendly more work could be done to make it look alike many well known chatting apps in market

Realtime Location

Made use og google maps Api to do this task and gained a great knowledge on how to implement Api's into an application. Group members realtime location can be traced if they allow it once asked for permission alert.

Grading System

Grading system is a Java based software used for the use of students and teacher to interac with each other online. The software is made with the data structure algoriths and also shows the use of ip address for the functional chatting from remote stations. Teachers could upload the student result into the system and are the system does the calculation for them as well allow them to chat with individual students. The students could see their grades and also chat with teacher for any queries.

User Login

Both teachers and students use the same approach for login. The radio buttons does the partition.

Student Login

Students is able to see his marks in both of his subjects and the system also calculates the GPA for the student. The right panel allows student to chat with the respective teacher for any queries to be made.

Teachers Login

Teacher could only see the class performance for his particular subject. Teacher is abe to see the marks and also could update it if required. The option to chat with student is given on the right panel and this is the place where Teacher would get any messages from students and is able to make clerifications.

Classic Cafe

Classic cafe is a website created during my software engineering course, with a focus to learn the use of spring framework and gain the experience of how the structure of programming works in an orgnization. It is a basic website created by use Java for the backend development and also has the implementation of the Spring MVC model. The users get registered upon signing up and are able to get the rewards after every 7 purchases that helps the company to make thier customers happy and increase their business.

Home Page

The Home page has a clear layout which make it very easy for the customers to interact with. The very middle part give the user with the information about the rewards.

Log In

Upon sign up, the user starts gaining rewards and gets to see it upon loggin in.

User rewards

The rewards are seen on the Home page. Every time a purchacse is made the purchase section goes up by 1. Upon completion of 7 purchases, the user gets a reward that could be used whenever is demanded.

Menu

Menu page has a cart made with the use of some made with the use of javascript and is very user friendly and interactive. Making the buying online experience very exciting.

Updated Rewards

After a reward or purchase gets increased, it gets updated in the database (made in MS Access), with the backend use of java. The rewards could be seen on the Home page upon refresh.

Softwares Used

The preparation of this website was made with the use of HTML, CSS, JAVASCRIPT (Software used was Visual Studio) for the front-end. Back-end was complied using Java (Software used was eclipse), a Maven project was created and the use of spring framework was used for getting the knowlegde of MVC structure. GITHUB was used to make contributions and understand to purpose of new branches and mergers that is a great part of to also learn about the agile environment. MS Access was used to create a SQL Database.
